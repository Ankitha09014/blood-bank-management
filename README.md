# Blood Bank Management System

## Overview

The Blood Bank Management System is a web-based application developed to streamline the management of blood donation and distribution activities. The system centralizes donor, recipient, hospital, and blood inventory information, enabling efficient blood availability tracking and reducing manual record-keeping.

The application improves the process of blood donation, request handling, inventory management, and blood distribution through a secure and user-friendly platform.

---

## Problem Statement

Traditional blood bank management often relies on manual processes, which can lead to delays, errors, and difficulties in tracking blood inventory. There is a need for a centralized system that can efficiently manage blood donors, recipients, hospitals, and blood stock information.

---

## Objectives

* Manage donor and recipient information.
* Maintain blood inventory records.
* Track blood donations and requests.
* Improve blood availability management.
* Reduce manual errors and processing time.

## Database Implementation

MongoDB is used to store and manage:

* Donor information
* Recipient records
* Blood inventory details
* Hospital requests
* User authentication data

The database enables efficient data retrieval, updating, and tracking of blood availability across the system.

---

## System Workflow

1. Users register and log in to the system.
2. Donors submit blood donation information.
3. Hospitals request required blood units.
4. Organizations manage blood stock and requests.
5. Admin monitors system activities and inventory.
6. Blood availability information is updated in real time.
